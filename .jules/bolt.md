## 2024-05-19 - [Mitigate N+1 Queries in TF-IDF scoring in Python Local RAG]
**Learning:** The initial implementation of `_compute_tfidf` retrieved document count and document frequencies for every iteration across a result set up to 700 conversations/notes (N+1 queries). This caused major bottlenecking (1.28 seconds to recall across a 1000 record test db). The local SQLite database memory uses `re.sub` and Python's string splitting which is inherently heavy when repeated frequently across 1000+ elements.
**Action:** When working on Python script data processing that iterates SQLite local databases, cache loop-invariant queries (e.g. `SELECT COUNT(*)`) ahead of the loop and implement a dict-based parameter cache (e.g., `doc_freq_cache`) to store specific term queries, mitigating repetitive per-document/per-term SQL executions.
