## 2024-05-18 - N+1 DB Queries in TF-IDF Implementation
**Learning:** The zero-dependency SQLite-backed TF-IDF semantic search (`ConversationMemory.recall()`) had a severe N+1 query problem. During vector calculation, it was executing a `SELECT count FROM doc_freq WHERE term = ?` and `SELECT COUNT(*) FROM conversations` for every individual token across all historical conversations during the scoring loop. On a database with only 500 records, a recall query could take 1.3 seconds due to thousands of repeated, individual SQLite queries.
**Action:** When implementing mathematical scoring across multiple rows in SQLite, always pre-load aggregate or frequency tables into memory (like a Python dictionary) before looping through rows. Passing the cached dictionary to the TF-IDF calculator reduced the recall operation time from 1.3 seconds to 0.2 seconds.
